# 故障排查指南

sci-group-read 技能的常见问题和解决方案。

## 阶段1：解析问题

### 问题：JSON解析错误

**症状：**
```
Error: Unable to extract text from PDF
```

**原因：**
- PDF是图像型（扫描文档）
- PDF已加密或有密码保护
- PDF文件损坏

**解决方案：**
1. 检查PDF是否基于文本：尝试手动从PDF复制文本
2. 如果是扫描版：使用OCR预处理（如Adobe Acrobat、Tesseract）
3. 如果已加密：先移除密码保护
4. 如果损坏：重新下载PDF

### 问题：缺少摘要或结论

**症状：**
- JSON文件的 `abstract` 或 `conclusion` 字段为空

**原因：**
- 非标准论文格式
- 摘要/结论未明确标注
- 特定页面的文本提取失败

**解决方案：**
1. 手动验证PDF是否包含这些章节
2. 检查章节是否使用非标准标题
3. 继续进行阶段2进行全文提取

### 问题：批量处理中途停止

**症状：**
- 只有部分PDF有JSON输出
- `batch_summary.json` 不完整

**原因：**
- 某个PDF导致错误
- 磁盘空间不足
- 进程超时

**解决方案：**
1. 检查错误日志找出具体失败的PDF
2. 单独处理该PDF查看详细错误
3. 跳过问题PDF继续处理其他文件
4. 确保有足够磁盘空间（至少1GB可用）

## 阶段2：提取问题

### 问题：章节边界不正确

**症状：**
- 章节合并在一起
- 章节在错误位置断开
- 缺少章节

**原因：**
- 复杂的PDF布局（多栏、特殊格式）
- 视觉模型误识别章节标题
- 非标准章节命名

**解决方案：**
1. 查看 `VISION_VERIFICATION_REPORT.md` 的置信度分数
2. 检查 `chapter_structure.json` 的检测章节
3. 如需要可手动调整章节边界
4. 对于复杂布局，考虑手动章节提取

### 问题：图片转换失败

**症状：**
```
Error: Unable to convert PDF to images
```

**原因：**
- 磁盘空间不足
- PDF渲染问题
- 缺少依赖项

**解决方案：**
1. 检查磁盘空间（每页需要约10MB）
2. 验证PDF能在PDF查看器中正常打开
3. 安装/更新所需库：
   ```bash
   pip install pdf2image pillow
   ```
4. 在macOS上，确保安装了poppler：
   ```bash
   brew install poppler
   ```

### 问题：文本提取不完整

**症状：**
- 章节有缺失文本
- 文本乱码或不正确
- markdown文件为空

**原因：**
- 图片质量差
- 复杂格式（公式、表格）
- 视觉模型限制

**解决方案：**
1. 检查原始PDF质量
2. 对于公式：可能需要手动LaTeX转录
3. 对于表格：考虑单独提取
4. 在 `pdf_to_pages.py` 设置中提高图片分辨率

### 问题：验证报告显示低置信度

**症状：**
- `VISION_VERIFICATION_REPORT.md` 有警告
- 置信度分数低于0.7

**原因：**
- 不寻常的论文结构
- PDF质量差
- 非标准格式

**解决方案：**
1. 手动检查提取的章节
2. 与原始PDF对比
3. 用调整后的参数重新提取
4. 如果内容正确，接受较低置信度

## 阶段3：分析问题

### 问题：分析过于肤浅

**症状：**
- `deep_analysis.md` 缺乏深度
- 缺少批判性评估
- 观察过于笼统

**原因：**
- 阶段2的章节提取不完整
- 缺少关键章节（方法论、结果）
- 上下文不足

**解决方案：**
1. 验证阶段2提取了所有章节
2. 检查 `full_text.md` 是否完整
3. 确保存在方法论和结果章节
4. 用完整内容重新运行分析

### 问题：分析生成失败

**症状：**
```
Error: Unable to generate analysis
```

**原因：**
- 缺少输入文件
- markdown文件损坏
- 内容不足

**解决方案：**
1. 验证论文目录结构：
   ```
   paper_Name/
   ├── sections/
   ├── full_text.md
   └── index.md
   ```
2. 检查markdown文件不为空
3. 确保 `full_text.md` 有实质内容

## 阶段4：领域综述问题

### 问题：领域综述不完整

**症状：**
- 报告遗漏论文
- 综述肤浅
- 无跨论文洞察

**原因：**
- 并非所有论文都完成了阶段3
- 缺少 `deep_analysis.md` 文件
- 论文数量不足

**解决方案：**
1. 验证所有论文都有 `deep_analysis.md`：
   ```bash
   find results/ -name "deep_analysis.md"
   ```
2. 确保至少有3-5篇论文才能进行有意义的综述
3. 为缺少分析的论文重新运行阶段3

### 问题：报告关注错误方面

**症状：**
- 综述与研究重点不匹配
- 缺少关键主题

**原因：**
- 混合了不同领域的论文
- 研究重点不明确

**解决方案：**
1. 在阶段4前按子领域组织论文
2. 对特定子目录运行 `/field`
3. 提供更清晰的研究重点上下文

## 通用问题

### 问题：性能缓慢

**症状：**
- 阶段耗时远超预期
- 系统无响应

**原因：**
- PDF文件过大
- 处理论文过多
- 系统资源不足

**解决方案：**
1. 分小批次处理论文
2. 关闭其他应用程序
3. 增加超时设置
4. 考虑大批量处理时过夜运行

### 问题：磁盘空间问题

**症状：**
```
Error: No space left on device
```

**原因：**
- 阶段2的图片文件占用大量空间
- 处理论文过多

**解决方案：**
1. 清理旧结果：
   ```bash
   rm -rf results/*/pages/  # 删除中间图片
   ```
2. 分批处理论文
3. 将完成的分析归档到外部存储

### 问题：权限错误

**症状：**
```
Error: Permission denied
```

**原因：**
- 文件权限不足
- 受保护的目录

**解决方案：**
1. 检查文件权限：
   ```bash
   ls -la results/
   ```
2. 确保对results目录有写入权限
3. 使用适当权限运行

## 获取帮助

如果问题持续存在：

1. 查看[工作流指南](workflow-guide.md)了解正确用法
2. 查看[脚本文档](scripts-guide.md)了解高级选项
3. 详细检查错误日志
4. 先用简单测试PDF尝试
5. 验证所有依赖项已安装

## 报告问题

报告问题时，请包含：
- 错误发生的阶段
- 完整错误消息
- PDF特征（大小、页数、格式）
- 系统信息（操作系统、Python版本）
- 重现步骤
