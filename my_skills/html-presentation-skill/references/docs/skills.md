# Markdown 格式规范

本文档定义 HTML 演示文稿的 Markdown 格式要求。

## 基本结构

```markdown
# 演示文稿标题

副标题（可选）
作者信息（可选）
日期（可选）

---

## 目录

1. 章节一
2. 章节二
3. 章节三

---

## 章节标题

正文内容...

---

## 结束页

致谢内容
```

## 元素语法

### 文本元素

| 语法 | 效果 | 示例 |
|------|------|------|
| `# 标题` | 封面大标题 | `# 技术分享` |
| `## 标题` | 幻灯片标题 | `## 核心概念` |
| `### 标题` | 小标题/分区 | `### 背景介绍` |
| `**文本**` | 加粗强调 | `**重要**` |
| `*文本*` | 斜体 | `*注意*` |
| `> 引用` | 引用块 | `> 关键观点` |

### 列表

```markdown
- 无序列表项 1
- 无序列表项 2
- 无序列表项 3

1. 有序列表项 1
2. 有序列表项 2
3. 有序列表项 3
```

### 表格

```markdown
| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 单元格 1 | 单元格 2 | 单元格 3 |
| 单元格 4 | 单元格 5 | 单元格 6 |
```

### 代码块

````markdown
```python
def hello():
    print("Hello, World!")
```
````

### 图表数据

支持自动识别可可视化的数据：

```markdown
### 销售数据对比

| 产品 | Q1 | Q2 | Q3 |
|------|-----|-----|-----|
| 产品 A | 100 | 150 | 200 |
| 产品 B | 80 | 120 | 180 |
| 产品 C | 60 | 90 | 140 |
```

### 媒体资源

```markdown
![描述文字](图片链接)

![视频封面](视频封面图链接)
@[video](视频链接)
```

## 特殊幻灯片类型

### 封面页

第一个 `#` 标题自动作为封面：

```markdown
# 主标题

副标题

作者：张三
日期：2026-03-25
```

### 目录页

标题包含"目录"或"议程"的页面：

```markdown
## 目录

1. 第一章
2. 第二章
3. 第三章
```

### 章节分隔页

```markdown
## 第一部分

### 背景与动机
```

### 结束页

最后一个幻灯片，标题包含"谢谢"、"结束"、"Q&A"等：

```markdown
## 谢谢聆听

Q&A
```

## 内容类型识别

系统会自动识别以下内容类型并选择最佳展示方式：

| 类型 | 识别特征 | 展示方式 |
|------|---------|---------|
| 文本 | 段落、列表 | 标准布局 |
| 表格 | Markdown 表格语法 | 表格布局 |
| 图表 | 数值对比数据 | 柱状图/折线图 |
| 代码 | ``` 代码块 | 代码高亮 |
| 混合 | 多种元素组合 | 分栏/卡片 |
| 时间线 | 日期 + 事件 | 时间轴布局 |

## 分页规则

- 使用 `---` 作为分页符
- 每个 `##` 标题开始新幻灯片
- 内容过多时建议手动分页

## 最佳实践

1. **每页一个核心观点** - 避免信息过载
2. **列表优于段落** - 更易扫描
3. **数据可视化** - 数值对比用图表
4. **代码精简** - 只展示关键片段
5. **图片适度** - 每页最多 1-2 张

## 执行 SOP 说明

### 核心原则

**本技能没有一键生成脚本**，所有流程必须由主 Agent  orchestrate，通过调用工具脚本和子 agent 协作完成。

### 工具脚本（仅作为工具使用）

| 脚本 | 功能 | 调用方式 |
|------|------|---------|
| `scripts/chunk_splitter.py` | 按 `---` 分页符拆分 Markdown | 主 Agent 直接调用 |
| `scripts/table_generator.py` | 生成 HTML 表格 | 在子 Agent 中通过 Python 调用 |
| `scripts/chart_generator.py` | 生成 SVG 图表 | 在子 Agent 中通过 Python 调用 |

### 主 Agent 职责

- **流程控制**：主 Agent 负责整个 SOP 的流程控制和用户交互
- **脚本调用**：主 Agent 直接调用工具脚本（如 chunk_splitter.py）
- **子 Agent 分发**：为具体任务（内容分析、渲染）创建子 Agent
- **最终组装**：主 Agent 收集所有子 Agent 返回的 HTML 片段，组装为完整 HTML

### 子 Agent 职责

- **内容分析**：分析 Markdown chunk，提取结构化信息
- **内容渲染**：根据内容类型（表格、图表、文本）渲染 HTML
- **格式审核**：审核 Markdown 是否符合规范

### 禁止事项

- 不允许使用一键生成脚本（如 build_presentation.py 已被删除）
- 不允许跳过用户确认步骤（如风格选择、内容确认）
- 不允许绕过异常处理（任何检测到的异常都必须与用户沟通）
